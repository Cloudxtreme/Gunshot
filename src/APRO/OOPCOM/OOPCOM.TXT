!TOPIC 112 Browser commands
!NOINDEX
When the Browser window is active, the status line will look something
like this:

<F3> New file <F9> Menu <AltH> Hex/Ascii <Esc> Close | Line 1331 Col 1

The numbers on the right indicate the number of the line at the top of the
window, and the column for the character displayed at the left edge of the
window. When the Browser is reading from the disk or searching, a
"Working..." message will be displayed in their place.

If you want to browse a different file, press <F3> and enter the name of
the file you wish to browse. To choose the file from a directory listing,
enter the appropriate file mask--e.g., 'C:\DOCS\*.DOC'--and press
<Enter>.

The following is a summary of keystrokes understood by the browser:

Cursor Movement

Left          <Left>, <CtrlS>
!LINE
Scroll window left 1 column.

Right         <Right>, <CtrlD>
!LINE
Scroll window right 1 column.

Word left     <CtrlLeft>, <CtrlA>
!LINE
Scroll window left 10 columns.

Word right    <CtrlRight>, <CtrlF>
!LINE
Scroll window right 10 columns.

Start of line <Home>, <CtrlQ><S>
!LINE
Scroll window to column 1.

End of line   <End>, <CtrlQ><D>
!LINE
Scroll window to leftmost column, so that the end of the longest line
on the screen is displayed.

Up            <Up>, <CtrlE>, <CtrlW>
!LINE
Scroll window up one line.

Down          <Down>, <CtrlX>, <CtrlZ>
!LINE
Scroll window down one line.

Page up       <PgUp>, <CtrlR>
!LINE
Scroll window up one page.

Page down     <PgDn>, <CtrlC>
!LINE
Scroll window down one page.

Top of file   <CtrlPgUp>, <CtrlQ><R>
!LINE
Scroll to beginning of file.

End of file   <CtrlPgDn>, <CtrlQ><C>
!LINE
Scroll to end of file.

Jump to line  <CtrlJ><L>
!LINE
Prompts the user for a line number, then scrolls the window such that
the specified line is at the top of the window. If the line number
specified is greater than the number of lines in the file, the window
will be scrolled to the end of the file.

File Commands

New file      <F3>, <CtrlK><N>
!LINE
Load a new file into the browser.

Block Commands

Begin block   <F7>, <CtrlK><B>
!LINE
Marks the line at the top of the window as the start of a block. Note
that the browser allows only whole lines to be part of a block.

Block end     <F8>, <CtrlK><K>
!LINE
Marks the line at the top of the window as the end of a block.

Block bottom  <CtrlB><K>
!LINE
Marks the line at the bottom of the window as the end of a block.

Jump begin    <CtrlQ><B>
!LINE
Jump to the beginning of the currently marked block.

Jump end      <CtrlQ><K>
!LINE
Jump to the end of the currently marked block.

Block toggle  <CtrlK><H>
!LINE
Toggle the display of marked blocks. Note that the ccBlkWrite and
ccBlkPrint commands both generate errors if a marked block is not
displayed.

Block write   <CtrlK><W>
!LINE
Writes the currently marked and displayed block to a file. Note that this
command writes the actual contents of the file being browsed, not the
image of it that is being displayed on the screen. The command does the
same thing whether in ASCII or hex mode.

Block print   <CtrlK><P>
!LINE
Writes the currently marked and displayed block to the printer. As with
the Block write command, this option writes the actual contents of the
file being browsed to the printer. This command directs its output to
LPT1.

Search

Search        <CtrlQ><F>
!LINE
Allows the user to search for any string of up to 30 characters. After
entering a search string, the user is prompted for search options, which
are handled in the same fashion as the Borland editors. Valid search
options are 'B' (search Backwards), 'G' (search Globally), and 'U' (ignore
case). If the search string is found, the line where the text is found
will be highlighted until another search operation is performed or the
mode is toggled between hex and ASCII.

Repeat search <CtrlL>
!LINE
Repeat the last search operation. The string used in the last search
operation will be used again, as will the previous search options. If no
search operation has yet been performed, this command does nothing.

Text Markers

Set marker    <CtrlK><0>..<CtrlK><3>
!LINE
Sets one of the ten text markers at the start of the line at the top of
the window. <CtrlK><0> sets marker 0, <CtrlK><1> sets marker 1, etc.
Supports 4 text markers, 0..3.

Jump marker   <CtrlQ><0>..<CtrlQ><3>
!LINE
Scrolls the window such that the line associated with the specified text
marker is at the top of the window. <CtrlQ><0> jumps to marker 0,
<CtrlQ><1> jumps to marker 1, etc.

Mode Toggles

Hex mode      <CtrlH>
!LINE
Toggles between ASCII mode (the default) and hex mode. In hex mode, either
8 or 16 characters are displayed per line, depending on the width of the
window. (If the inner portion of the window is <= 50 columns 8 characters
are displayed per line). The leftmost column displays the offset within
the file (in hex) for the first character in the "line," the middle
columns display the contents of the line in hex, and the rightmost column
displays the contents of the line in ASCII.

Strip high    <CtrlQ><H>
!LINE
Activates or deactivates the option to suppress (strip) the eighth (high)
bit for each character in the file. In ASCII mode, this option affects
only characters with an ordinal value of 128 or higher (10000000 in
binary), causing #228, for example, to be displayed as #100 ('d'). In hex
mode, it affects control characters as well, causing them to be
represented as periods ('.'). The hex values in the middle columns are not
affected by this option.

Tab expand    <CtrlQ><T>
!LINE
Enables or disables tab expansion. When this option is on, tab characters
(^I) are expanded to spaces, aligning the text that follows at the next
tab stop. (Tab stops occur every 8 columns: 1, 9, 17, etc.) This option
does nothing in hex mode.

Other

Quit          <Esc>, <ClickRight>
!LINE
Quit browsing.

Help          <F1>, <ClickBoth>
!LINE
Help. Brings up context sensitive help system.

If use of the mouse is enabled, you may use the mouse to scroll vertically
and horizontally by moving the scroll bars.

!TOPIC 2 Terminal window
The Terminal window is a moveable, resizeable window that displays
incoming and outgoing characters. The terminal window supports a scroll
back buffer of 0-400 lines (configured by OOPINST). The terminal also

the scroll back buffer to a file.

!TOPIC 1 Dialer
Phone book viewer and editor. Lets you dial a number in the phone book
at the touch of a key.

The Dialer window is used to display and edit phone books--data files
that store phone numbers and communications parameters--and to dial
numbers stored in phone books for you.

See also:  0114Dialer commands  0002Terminal window
;------------------------------------------------------------------------
!TOPIC 114 Dialer commands
!NOINDEX
To edit an existing phone book entry, move the highlight bar over it and
press <Enter>. The same data entry screen is used to edit entries that
is used to create them. To delete a phone book entry, move the highlight
bar over it and press <CtrlY>.

To dial a phone number, move the highlight bar over the number and press
<AltD>. The Dialer will then try to dial the selected number using the
communications parameters specified in the selected phone book entry. If
a successful connection is made, the Dialer window will close itself
automatically and either open the Terminal window or make it the current
window, as appropriate.

See also:  0001Dialer           0111Global commands

!TOPIC 123 Dos shell
Selecting this menu option causes OOPCOM to execute a second copy of
COMMAND.COM, giving you an opportunity to run other programs or execute
DOS commands.

This facility may be used even if a communications session is in progress.
If you have the Terminal window open and you shell to DOS, when you return
to the program the Terminal window will display the characters that were
received while you were shelled out. Be aware, however, that characters
can be lost if OOPCOM's 2K buffer for incoming characters becomes full
and the software flow control option is not enabled.
